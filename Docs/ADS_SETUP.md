# Utawave 広告枠 セットアップガイド

起動画面の広告枠を**有効化して配布する人向け**の手順書です。
（コードの内部実装は `CLAUDE.md`「起動画面の広告枠」を参照）

---

## 1. 概要 — 広告は既定でオフ

- **公開ソースをそのままビルドすると広告は出ません**（起動画面は従来どおり 2 列・通信なし）。
- 広告は **ビルド時のフラグ** で有効化します。フラグを立てたビルドだけが 3 列の広告枠を表示し、
  指定した URL から広告データ（JSON）を取得します。
- これにより、ソースを公開しても「誰がビルドしても広告が出る」状態にはなりません。
  広告を出すのは**公式配布ビルド（あなたが CI で作るビルド）だけ**です。

### 有効/無効を決める 2 つのスイッチ

| スイッチ | 種類 | 既定 | 役割 |
|---|---|---|---|
| `UTAWAVE_ADS_ENABLED` | **ビルド時**（CMake） | OFF | 広告機能のマスタースイッチ。OFF だと広告コードは一切動かない |
| `showAds` | 実行時（環境設定） | ON | 公式ビルド内で**各ユーザーが**広告を切るためのトグル |

実際に広告が表示される条件は **`UTAWAVE_ADS_ENABLED` が ON かつ `showAds` が ON** の両方です。
公開ソースの通常ビルドは `UTAWAVE_ADS_ENABLED` が OFF なので、環境設定にも広告トグルは出ません。

---

## 2. ビルド時に渡すフラグ

| CMake 変数 | 例 | 説明 |
|---|---|---|
| `UTAWAVE_ADS_ENABLED` | `ON` | 広告枠を有効化（3 列表示） |
| `UTAWAVE_AD_FEED_URL` | `https://example.com/ads/feed.json` | 広告 JSON の取得先 URL（実行ファイルに埋め込まれる） |

`UTAWAVE_AD_FEED_URL` を省略すると、ソース内のプレースホルダ URL
（`https://utawave.com/ads/feed.json`）が使われます。本番では必ず自分の URL を指定してください。

---

## 3. ローカルで有効化してビルドする

```bash
# 設定（広告 ON ＋ 取得先 URL を指定）
cmake -S . -B build \
      -DUTAWAVE_ADS_ENABLED=ON \
      -DUTAWAVE_AD_FEED_URL="https://example.com/ads/feed.json"

# ビルド
cmake --build build --target Utawave --config Release
```

configure 時に次のようなログが出れば有効化できています:

```
-- Utawave: startup ad panel ENABLED
-- Utawave: ad feed URL = https://example.com/ads/feed.json
```

### 元に戻す（広告オフのビルドに戻す）

`build/` の CMake キャッシュにフラグが残るので、明示的にオフへ戻します:

```bash
cmake -S . -B build -DUTAWAVE_ADS_ENABLED=OFF -DUTAWAVE_AD_FEED_URL=""
```

### 開発用: サンプル広告で見た目を確認する（サーバー不要）

本番サーバーがまだ無い段階で広告枠の見た目（2 件ピーク表示の縦スクロール・自動送り・クリック・
画像なしの表示など）を確認したいときは、`UTAWAVE_ADS_DEBUG` を使います。広告枠を有効化したうえで、
**通信せず内蔵のサンプル広告（30 件・画像つき）を用意**し、そこから毎回ランダムに 15 件を表示します
（ランダム抜粋・件数の見え方の確認用）。

```bash
# 設定（デバッグサンプルモード）
cmake -S . -B build -DUTAWAVE_ADS_DEBUG=ON

# Debug でビルド
cmake --build build --target Utawave --config Debug

# 実行（macOS）
open build/Utawave_artefacts/Debug/Utawave.app
```

configure 時に次のログが出れば OK:

```
-- Utawave: startup ad panel ENABLED (debug sample mode / no server)
```

- `UTAWAVE_ADS_DEBUG=ON` は **自動的に広告枠を有効化**します（`UTAWAVE_ADS_ENABLED` を別途付ける
  必要はありません）。
- 表示されるのは `AdService::sampleAds()` が返す**ダミー広告**です（バナー画像はコード内で動的生成）。
  内容を変えたい場合は `Source/Project/AdService.cpp` の `sampleAds()` を編集します。
- Debug でなく Release でビルドしても、フラグが付いていればサンプル広告が出ます
  （フラグはビルド構成ではなく configure で決まります）。
- **本番ビルドにこのフラグを付けないこと**（サンプル広告が出てしまいます）。確認が終わったら
  上の「元に戻す」に加えて `-DUTAWAVE_ADS_DEBUG=OFF` を付けて再 configure してください:
  ```bash
  cmake -S . -B build -DUTAWAVE_ADS_ENABLED=OFF -DUTAWAVE_ADS_DEBUG=OFF -DUTAWAVE_AD_FEED_URL=""
  ```

> 実際のネットワーク取得（本番 JSON / 画像 DL / キャッシュ）を Debug で試したい場合は、
> サンプルではなく `-DUTAWAVE_ADS_ENABLED=ON -DUTAWAVE_AD_FEED_URL="http://localhost:8000/feed.json"`
> のように**実 URL** を指定してください（例: `python3 -m http.server` でローカルに JSON と画像を配信）。

## 4. GitHub Actions で有効化する（推奨）

URL をソースに直書きせず、CI のビルド時に注入します。

### 4-1. URL をリポジトリ変数に登録

GitHub リポジトリの **Settings → Secrets and variables → Actions → Variables** で
新しい変数を作成します（URL は秘密情報ではなく実行ファイルに埋め込まれるため、Secret ではなく
**Variable** で十分です。社内方針で隠したい場合は Secret でも可）。

- 名前: `UTAWAVE_AD_FEED_URL`
- 値: `https://example.com/ads/feed.json`

### 4-2. ワークフロー例

```yaml
name: Release (ads enabled)

on:
  workflow_dispatch:        # 手動実行（公式配布ビルド用）
  push:
    tags: ['v*']            # タグ push でリリースビルド

jobs:
  build:
    strategy:
      matrix:
        os: [macos-latest, windows-latest]
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
        with:
          submodules: recursive   # JUCE をサブモジュール管理している場合

      - name: Configure (ads ON)
        run: >
          cmake -S . -B build
          -DCMAKE_BUILD_TYPE=Release
          -DUTAWAVE_ADS_ENABLED=ON
          -DUTAWAVE_AD_FEED_URL="${{ vars.UTAWAVE_AD_FEED_URL }}"

      - name: Build
        run: cmake --build build --target Utawave --config Release

      # 以降、署名・公証・アーティファクト化など既存のリリース手順を続ける
```

ポイント:
- **公式リリースのワークフローだけ** `-DUTAWAVE_ADS_ENABLED=ON` を付けます。
  通常の CI（テスト用など）はフラグを付けなければ広告オフのままです。
- `vars.UTAWAVE_AD_FEED_URL` が空のままだとプレースホルダ URL になり広告は出ません。
  リリース前に変数の登録を確認してください。

## 5. 広告フィード（JSON）の仕様

`UTAWAVE_AD_FEED_URL` が返す JSON の形式です。サーバー側で用意します。

### スキーマ

```jsonc
{
  "ads": [
    {
      "id":       "spring2026",                       // 任意（識別用）
      "title":    "新発売プラグイン",                   // 見出し（カードに表示）
      "body":     "ボーカルに特化した高品質リバーブ",    // 任意・予備（現在は表示しません）
      "imageUrl": "https://example.com/banner.png",   // バナー画像 URL（カードに表示）
      "linkUrl":  "https://example.com/lp",           // クリック時に開く URL（任意）
      "lang":     "ja",                               // 任意・対象言語 "ja"/"en"/省略・"all"（後述）
      "pinned":   true                                // 任意・true で常にトップ固定（月1件の特集バナー・後述）
    },
    {
      "id":       "campaign-x",
      "title":    "期間限定キャンペーン",
      "body":     "今だけ◯◯",
      "linkUrl":  "https://example.com/x"
    }
  ]
}
```

- トップレベルが `{ "ads": [ ... ] }` でも、**配列そのもの** `[ { ... }, ... ]` でも受け付けます。
- 各フィールドはすべて文字列。**`title` / `body` / `imageUrl` のいずれか 1 つ以上**が無い項目は
  無視されます（表示するものが無いため）。
- **表示されるのは「バナー画像 + 見出し（`title`）」だけ**です（シンプルさ優先）。`body` は今は
  表示しません（互換性のため受け付けますが将来用の予備）。実質は **`imageUrl` と `title`** を用意します。
- `linkUrl` が無い広告はクリックしても何も起きません。
- フィードは**最大 30 件**まで読み込みます（31 件目以降は無視）。**そのうち実際に表示するのは 15 件**で、
  **起動ごとにランダムに 15 件を抜粋**します（30 件登録しても、毎回違う 15 件が出ます。登録順には依存しません）。

### フィールド詳細

| フィールド | 必須 | 説明 |
|---|---|---|
| `id` | 任意 | 任意の識別子。表示には使いません |
| `title` | ※ | 見出し（バナー下に 1 行・はみ出しは省略） |
| `imageUrl` | ※ | バナー画像。PNG / JPEG / GIF。`https` 推奨 |
| `body` | 任意 | 現在は**非表示**（予備フィールド） |
| `linkUrl` | 任意 | クリックで OS の既定ブラウザが開く URL |
| `lang` | 任意 | 対象言語 `"ja"` / `"en"` / 省略・`"all"`（=全言語向け）。詳細は下の「多言語・海外展開」 |
| `pinned` | 任意 | `true` で**常にトップ固定**（ランダム表示の対象外）。詳細は下の「固定バナー（月1件）」 |

※ `title` / `imageUrl` は最低どちらか 1 つを推奨（両方あると一番きれい）。

### 画像の推奨

| 項目 | 推奨 |
|---|---|
| **サイズ** | **600 × 300 px（アスペクト比 2:1）** |
| 最低 | 400 × 200 px |
| 比率 | **2:1 を基準**（1.8:1〜2.2:1 程度） |
| 形式 | **PNG / JPEG**（透過は PNG） |
| 容量 | 1 枚 **~150KB 以内**が目安 |

- **実際の表示領域は約 200 × 95px（論理px・広告 3 件以上のとき）≒ 2:1** です。画像は幅いっぱいに
  合わせ、はみ出した上下を自動でトリミングします（cover）。高解像度ディスプレイ（Retina = 2 倍）で
  くっきり見せるため、表示領域の 2 倍以上で用意します（600 × 300 なら余裕があり将来も流用しやすい）。
- **比率は 2:1 が最適**。これより横長（例 3:1）にすると**左右が切れ**、縦長にすると上下が切れます。
- cover で上下が切られることがあるので、**ロゴ・文字・重要な要素は中央寄せ**にすると安全です。
- 見出し（`title`）は画像の**下に別テキストで表示**されるので、バナーに文字を焼き込まなくても OK です。
- 表示件数で領域の高さが少し変わります（2 件のみ＝やや縦長 約 1.6:1／3 件以上＝約 2:1）。
  **2:1 + 中央寄せ**ならどの件数でも安全です。
- **登録した全バナー（最大 30 枚）を起動時にダウンロード**するため、1 枚は軽めに（合計で数 MB 以内が理想）。

### 多言語・海外展開（言語別配信）

アプリの言語設定が**日本語なら `ja`、それ以外（英語など）は英語圏として `en`** の広告を表示します。
配信方法は **2 通り**あり、どちらでも（併用でも）動きます。

**方法 A: 言語別フィード（推奨・静的ホスティング向き）**
フィード URL に `{lang}` を入れておくと、アプリが起動時に `ja` / `en` へ置き換えてリクエストします。

```bash
# CI で URL に {lang} を含める
-DUTAWAVE_AD_FEED_URL="https://example.com/ads/{lang}.json"
```
→ 日本語ユーザーは `.../ads/ja.json`、それ以外は `.../ads/en.json` を取得します。
`ja.json` と `en.json` をそれぞれ用意してください（各最大 30 件）。

**方法 B: 1 つのフィードに混在＋ `lang` で振り分け**
1 つの JSON に全広告を入れ、各広告に `lang`（`"ja"` / `"en"` / 省略・`"all"`）を付けます。アプリが
**現在の言語に一致する広告＋全言語向け（`lang` 省略 or `"all"`）だけを表示**します。

```jsonc
{ "ads": [
  { "title": "国内向け広告", "imageUrl": "…", "lang": "ja" },
  { "title": "Overseas ad",  "imageUrl": "…", "lang": "en" },
  { "title": "どちらにも出す", "imageUrl": "…" }            // lang 省略 = 全言語
] }
```

- `{lang}` を使わない URL の場合は、アプリが自動で **`?lang=ja` / `?lang=en` をクエリに付与**します。
  動的サーバーならこれで言語を判定できます。静的ホストはクエリを無視するので、その場合は方法 B（`lang`
  フィールド）でクライアント側が振り分けます。
- **件数の上限は言語ごとに 30 件**です（方法 A なら各ファイル 30 件まで。方法 B の混在フィードは
  「先頭 30 件をパース → 言語で抽出」のため、**1 ファイルに大量に混ぜるより方法 A を推奨**します）。
- キャッシュは言語別（`ads_ja.json` / `ads_en.json`）に分かれ、言語を切り替えても各言語の直近広告が残ります。

### 固定バナー（月1件・トップ固定）

特集したい広告を**常にトップ（先頭）に固定**できます。広告に `"pinned": true` を付けるだけです。

- `pinned` を付けた広告は**ランダム抜粋・シャッフルの対象外**になり、起動時に常に**1 件目**に表示されます。
- 「月 1 件」はフィードの運用で回します（その月の特集に `pinned` を付け、翌月は別の広告へ付け替える）。
  アプリ側に月次の状態保存は不要です。
- `pinned` は**言語フィルタの後**に並ぶので、出したい言語に合わせて `lang` を一致させてください
  （`lang` 省略 / `"all"` なら全言語で出ます）。例: 国内特集は `{ "pinned": true, "lang": "ja", … }`。
- `pinned` も表示上限（15 件）の 1 枠を使います（pinned 1 + ランダム 14）。複数付けた場合は配信順のまま
  先頭に並びます。

## 6. 動作仕様（有効時）

- **配置**: 起動画面の「最近使ったプロジェクト」の右に独立した列（ウィンドウ幅は 820 → 1080 に拡張）。
- **カード**: バナー画像 + 見出しのシンプルな構成。
- **2 件ピーク表示の縦スクロール**: 常に **2 件**を表示し、上下に隣の広告が少し**見切れて覗く**ように
  1 枚ずつ流れます。
  - 自動送り: 約 **4 秒**ごとに次へスクロール（マウスを乗せている間は一時停止）。
  - 手動: マウスを乗せると上辺・下辺の**中央に出る上下のシェブロン（▲▼）**、または **マウスホイール**で切替。
  - 広告が **3 件以上**でスクロールします（2 件はそのまま固定表示、1 件は中央表示）。
- **ページ位置ドット**: **下部に横並び**で、表示中の広告の**件数**と「いま見えている 2 件」が分かります。
  ドットをクリックするとその広告へジャンプします（表示は最大 15 件なのでドットも最大 15 個）。
  シェブロンと同じく**マウスを乗せた時だけ表示**され、何もしないと隠れます。
- **読み込み（最新優先）**: 起動時はまず「広告を読み込み中…」を表示して**最新の取得を待ちます**。
  取得できたら最新を表示し、**取得できなかった時だけ前回のキャッシュ（直近の広告）にフォールバック**します
  （キャッシュも無ければ「読み込めませんでした」）。
- **固定バナー優先 + ランダム抜粋**: `pinned` の広告を**常にトップに固定**し、残りはフィードに最大 30 件
  登録できるうち**ランダムに（合計）15 件**を抜粋・並び替えして表示します（特定の広告だけが常に出続けない
  ように。`pinned` は例外で常にトップ）。
- **言語で出し分け**: アプリの言語が日本語なら国内（`ja`）、それ以外なら英語圏（`en`）の広告を表示します
  （詳細は「5. 多言語・海外展開」）。
- **クリック**: 表示中の広告をクリックすると、その広告の `linkUrl` を OS の既定ブラウザで開きます。
- **取得**: 起動時にバックグラウンドで JSON を取得（接続タイムアウト 5 秒）。UI はブロックしません。
- **キャッシュ**: 直近に取得できた広告と画像を端末に保存し、次回起動時はまず即時表示 → 裏で最新へ更新。
  オフライン時は前回の広告が出ます。
  - 保存先: macOS `~/Library/Application Support/Utawave/AdCache/`、
    Windows `%APPDATA%\Utawave\AdCache\`
- **データが無い / 取得失敗時**: 「広告を読み込めませんでした / インターネット接続を確認してください」
  と **再読み込み**ボタンを表示します（= 仕様どおりのフォールバック。サーバー未稼働時もこれが出ます）。

## 7. ユーザー側の個別オフ

公式ビルド（広告有効）では、エンドユーザーが
**環境設定 → 「起動画面」→「起動画面に広告を表示する」** で広告を切れます（OFF にすると
広告列を出さず通信もしません）。次回起動画面表示時に反映されます。
公開ソースの通常ビルドではこのトグル自体が表示されません。

## 8. リリース前チェックリスト

- [ ] 広告 JSON を返すサーバー（`UTAWAVE_AD_FEED_URL`）が稼働している
- [ ] JSON が「5. 仕様」の形式・文字コード（UTF-8）で返る
- [ ] 画像 URL（`imageUrl`）が `https` で到達でき、表示できる形式（PNG/JPEG）である
- [ ] `linkUrl` が正しいランディングページを指している
- [ ] CI（GitHub Actions）の変数 `UTAWAVE_AD_FEED_URL` が登録済み
- [ ] リリースワークフローで `-DUTAWAVE_ADS_ENABLED=ON` が付いている
- [ ] 通常 CI（テスト等）にはフラグを**付けていない**（広告オフのまま）
- [ ] 配布ビルドで起動画面に 3 列・広告が表示されることを実機確認

## 9. トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| 広告枠が出ない（2 列のまま） | `UTAWAVE_ADS_ENABLED=ON` でビルドできていない。configure ログの `ad panel ENABLED` を確認 |
| 「広告を読み込めませんでした」 | URL に到達できない／JSON が壊れている／配列が空。URL とレスポンスを確認 |
| 画像が出ずテキストだけ | `imageUrl` が到達不能 or 非対応形式。`https` の PNG/JPEG にする |
| 古い広告が出続ける | 最新の取得に失敗し、キャッシュ（前回の広告）にフォールバック中。サーバー / ネットワーク / URL を確認。手動でリセットするなら `AdCache/` を削除 |
| クリックしても何も起きない | その広告に `linkUrl` が無い |
